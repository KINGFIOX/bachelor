一、课题背景

RISC-V 指令集架构的蓬勃发展

  • 2010 年由加州大学伯克利分校提出，采用开放源代码许可
  • 设计理念：精简、模块化、可扩展
  • RISC-V International 已拥有超过 4000 家会员单位
  • 应用覆盖从嵌入式微控制器到高性能服务器的完整生态
打破传统商业指令集架构（x86、ARM）的授权壁垒，为自主可控芯片提供可行技术路线

一、课题背景

产业界的广泛采纳

  • 乐鑫科技 ESP32-C3/C6 系列 — 物联网终端设备
  • NVIDIA 已部署超过 10 亿颗 RISC-V 核心用于片上控制
  • 工信部 2023 年成立 RISC-V 工作委员会，推动规模化应用
  • 国家战略：2025 年八部门联合发布指导意见，鼓励推广使用开源 RISC-V 芯片

一、课题背景

乱序执行 — 提升指令级并行度的关键

  • 1967 年 Tomasulo 提出动态调度算法，是现代高性能处理器的核心支柱
  • 在不违反程序语义的前提下，重新排列指令的执行顺序
  • 有效利用功能单元的空闲周期，显著提升流水线吞吐率

三、国内外研究现状

硬件描述语言与验证方法

  • Chisel：基于 Scala 的硬件构造语言
    • 与 Verilog 同属 RTL 层级，经过 circt 编译生成 Verilog
    • 更强的类型安全、参数化生成、模块复用
    • 软件工程方法应用于硬件设计
  • 多层次验证体系：
    • 模块级：UVM 受约束随机验证
    • 系统级：riscv-arch-test 指令集合规性
    • 差分测试：Difftest — RTL 与参考模型逐指令对比

四、现有研究的不足

1. 教学级与工业级乱序处理器之间存在显著断层

BOOM/香山复杂度极高（~15 万行），简单五级流水又未涉及乱序机制，中间缺少进阶参考设计

2. Chisel 生态下可复用 IP 核积累不足

SoC 外设 IP 核仍以 Verilog/VHDL 为主，需通过 BlackBox 封装， 无法发挥 Chisel 与 Diplomacy 优势 ：参数化、模块复用、类型安全
